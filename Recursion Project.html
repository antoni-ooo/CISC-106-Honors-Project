Recursion

is a peculiar function-writing paradigm that asks programmers to do the unintuitive when designing functions: to declare a function call to the very function they're defining, within said definition!

Although paradoxical on its face, recursion is a powerful tool in a programmer's tool-belt that allows one the ability to navigate data structures, and in our case, construct very cool fractal graphics in a way that would be difficult to achieve by other means! :]

CASE STUDY 3:

Tree

This graphic is a popular recursive graphic, but is surprisingly complicated to achieve in DrRacket due to the nature of how these shapes and images are created.

Despite this, we will want to begin this task as we would with any other recursive function: we want to account for a base case.

For our purposes of constructing a tree, we will have our base case return us a "leaf" that is entirely just a solid, green rectangle as our means of indicating completion:

Additionally, we're going to change the template of our function parameters: we're going to account for a "num" and for a "length".

We'll create some internal definitions within the function that give us the arithmetic needed to construct trees in such a way that they do not deconstruct (positioning in DrRacket can be tricky for these kinds of things. If recursive rectangles are too big in this function they will cause the tree to quite literally break apart and deform, hence the need for "new-length")

Considering that this function will be retuning our "leafs" upon num = 1, we have our function build our tree by dynamically creating solid, brown rectangles of new-length lengths.
